# DEC-0049 - Desktop-First, Mobile-Ready Platform Strategy

## Status

Approved by the Product Owner on 2026-07-24.

## Value

Both product and architecture.

## Impact

High.

## Decision

Family Memory AI will follow a **desktop-first, mobile-ready** strategy.

The Windows desktop application remains the only active implementation target until its core workflow is validated on a representative real family photo library. Complete desktop and mobile applications will not be developed in parallel.

All new implementation work must keep reusable business and domain logic independent from the PySide6 presentation layer wherever practical. Android is the planned second client of the Family Memory Engine and will begin as a focused companion application after the desktop workflow and reusable core boundaries are sufficiently validated.

## Permanent rules

- PySide6 is a presentation layer, not the product core.
- Business decisions and reusable domain logic must not live in UI widgets.
- Core services should use platform-neutral inputs and outputs wherever practical.
- Windows-specific file and photo access must remain behind platform boundaries.
- Photo, person, profile, album, and decision records require stable identifiers.
- Persistence formats must remain portable or explicitly migratable.
- Core separation is incremental and evidence-driven; no large mobile-preparation rewrite is approved.
- Cloud upload must not become mandatory merely to support mobile.
- Android begins only after the desktop workflow is useful, reliable, and sufficiently UI-independent.

## Delivery sequence

1. Validate the Windows desktop product.
2. Continue incremental reusable-core separation during normal desktop work.
3. Build a limited Android companion MVP.
4. Add desktop/mobile exchange for decisions, profiles, album state, and selected metadata.
5. Consider greater mobile independence only after the companion workflow is stable.

## Explicitly out of scope now

- full desktop and Android development in parallel;
- immediate rewriting of proven Python logic in Dart;
- one-for-one reproduction of desktop screens on mobile;
- mandatory cloud storage or original-photo upload;
- first-release processing of the complete historical 50,000-photo library on a phone;
- simultaneous Android, iPhone, Windows, and web development.
